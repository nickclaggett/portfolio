Problem Statement:

Users on Coursera often struggle to find the courses and learning materials they need due to limitations in the search functionality. Coursera's search results can be overwhelming, with less intuitive filtering, ranking options, and irrelevant search results, leading to frustration and decreased user satisfaction. Users report difficulty in narrowing down relevant courses, especially when searching for niche or specific topics. This impacts the overall user experience, reduces course enrollments, and hampers Coursera's ability to retain learners who value efficient, targeted search capabilities.

Feature Explanations

1. Search Filters

  • Availability of filters (e.g., price, difficulty level).

2. Ability to Filter Immediately

  • Ability to enter parameters upon the first search.

3. Search Bar Suggestions

  • Auto-suggestions for search queries as you type.
  • Related course or topic recommendations.
  • Predictive text to help refine search terms.

4. Keyword Search Accuracy

  • Effectiveness of keyword matching.
  • Search results showing relevant courses based on synonyms or closely related topics.
  • Support complex search queries (e.g., using multiple keywords).

5. Search by Categories or Skills

  • Ability to search by specific skill or learning path.
  • Pre-defined categories (e.g., business, data science, design) that users can browse and refine.
  • Skills-based search for users looking to develop expertise in a particular area.
  • Specific product/software skills like Excel, Tableau, PowerBi etc.

6. Search by Certification Type

  • Ability to filter by certification type (e.g., professional certificates, degrees, or casual learning).

7. Search by Partner or Institution

  • Search results that allow users to filter courses by specific universities, companies, or instructors.

8. Search Result Sorting

  • Ability to sort by relevance, popularity, ratings, newest, or enrollment numbers.

9. Search by Duration

  • Filters for course length (e.g., short courses, multi-week courses).
  • Ability to filter by total time commitment or weekly hours required.

10. Real-time Search Updates

  • Dynamic updates of results as filters or search terms are applied without reloading the page.
  • Smooth user experience when adjusting search parameters.

11. Search in Multiple Languages

  • Ability to search for courses in multiple languages, with language-specific search results.
  • Language-based filters or separate search options for non-English speakers.

12. Search Results Preview

  • Ability to preview course details (e.g., instructor bio, course overview, rating) without leaving the search results page.
  • Thumbnails or short video previews of the course content.

13. Search by Course Availability

  • Option to search for upcoming courses or currently available/open courses.
  • Filter for newly added courses or courses starting soon.

14. Search by Learning Style or Format

  • Ability to filter courses based on learning style preferences (e.g., video-based, text-based, interactive labs, projects).

​​15. Search by Enrollment Type

  • Filter between open enrollment (anytime access) and cohort-based courses, where users must enroll by a specific date.
  • Options to find self-paced or timed sessions with deadlines

16. Search by Job Progression

  • Ability to search job titles and have relevant courses appear

Review Points

  • Filters: Filter options are not immediately available when searching for a course. While this feature seems limited industry-wide, investigating this functionality may be beneficial. The idealistic options would be similar to house hunting websites where the searcher can put their parameters and then receive search results
  • Filters: If I click on one of the canned filter options while searching for courses to aid in learning french, the search results are irrelevant.
    • Ex: Search = Learn French
    • Canned Filter = Business
    • Results
    • The irrelevant search results found under these parameters can be replicated with any of the canned filters
  • Language Options: Changing the language of the website from French to English results in French results, however, the courses recommended are still irrelevant.
    • Ex. “Learn French” still brings me Spanish, English and Mandarin courses.
    • Ex. “Apprendre Francais” brings two relevant courses, the rest are unrelated to my search
  • Search Quality: When searching “Learn French” only two French courses are available - then subsequent results are “Covid-19 Education”, “Introduction aux droits de l’hommes”, and “Learn Mandarin”. If there are no other courses available these results should be separated similar to how UDemy has structured it
  • Search Result Preview: A searcher can view course ratings and the title, but they cannot view any of the content or a course overview - only “skills you will gain”. Udemy has a pop-up [that explains what you will learn] when your mouse hovers over the search result

Possible Options

Based on a direct feature comparison and user feedback regarding the search functions of Coursera and its competitors, Coursera could implement the following options to enhance its search functionality. Each option is plotted below according to its implementation difficulty and associated risk - considering whether the expected benefits will likely outweigh the time, labour, and advertising costs.

  1. Introduce filter options to narrow search results instantly.
  2. Enhance the tagging feature to display more accurate, relevant results.
  3. Provide a course preview to highlight key learning outcomes for potential learners.
  4. Clearly distinguish search results that match user queries from unrelated content when limited relevant options exist.
  5. Add course ratings to help users assess content quality and relevance.
  6. Offer skill progression recommendations within search results to guide users on their learning path.
